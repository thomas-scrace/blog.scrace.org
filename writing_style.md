# Writing style

The guide for writing posts on this blog.

Modern English, especially written English, is full of bad habits which spread
by imitation and which can be avoided if one is willing to take the necessary
trouble.

1. Never use a metaphor, simile, or other figure of speech which you are used to
   seeing in print.
2. Never use a long word where a short one will do.
3. If it is possible to cut a word out, always cut it out.
4. Never use the passive where you can use the active.
5. Never use a foreign phrase, a scientific word, or a jargon word if you can
   think of an everyday English equivalent.
6. Break any of these rules sooner than say anything outright barbarous.

These rules sound elementary, and so they are, but they demand a deep change of
attitude in anyone who has grown used to writing in the style now fashionable.

George Orwell, "Politics and the English Language" (1946)

## House rules

Follow these along with Orwell's rules above.

- Never use an em-dash (—). Where one would go, use a comma, a colon,
  parentheses, or a new sentence instead.
- Write in full, proper English sentences. No fragments, even for rhetorical
  effect. Headings may be short; body prose may not.
- Be matter-of-fact. This is technical writing, not a sales pitch: no
  sensational claims, no verdicts dressed up as punchlines, no manufactured
  drama.
- Be specific. Name the thing itself ("a pre-built index of the codebase"),
  not a vague class of things ("any preparation").
- Write for a technical reader. Do not explain basics (an index, a function,
  tokens) or reach for analogies where the plain term will do. Orwell's rule
  against needless jargon still applies.
- Prefer the plain phrase to the figurative one ("tasks I use agents for day
  to day", not "tasks I would hand over").
